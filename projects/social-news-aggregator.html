Revitalizing a Social News Aggregator

Project summary

A project that leveraged SQL DQL, DDL, and DML to create an improved database schema for a social news aggregator. The project specifically targets the enhancement of storage for forum posts, discussions, topics, and topic votes, followed by the transfer of existing data into the refined schema. Skills used include CREATE, ALTER, SELECT, INSERT, UPDATE, Constraints, Joins, Creating Views, CTE's, and Aggregate functions.

Skills used

  • DDL commands
    • CREATE TABLE
    • CREATE INDEX
    • ALTER TABLE
  • DML commands
    • SELECT
    • INSERT
    • UPDATE
  • UNIQUE, PRIMARY KEY, FOREIGN KEY, and CHECK constraints
  • DQL commands
    • Left Joins, Inner Joins and Self Joins
    • Creating Views
    • CTE's
    • Aggregate Functions

Data source

The dataset was provided as part of Udacity's SQL Nanodegree program.

Project goal

To kick off this Udacity SQL Nanodegree project, I received a dataset containing user information, user posts, votes on user posts, and user comments, organized into two distinct tables. The project goal was to:

  1. Investigate the existing schema
  2. Create the DDL for a new schema
  3. Migrate the existing data into the new schema using DML
  4. Using DQL to query the new schema, address a set of prompts from Udacity to understand user behavior, comments on
  5. posts, and votes on posts.

Project reflection

Prior to this course, my primary strength lay in Excel. During this project, I struggled with not being able to see and navigate data as effortlessly in SQL as I could in Excel. Although I found the DML portion of the project to be straightforward, I often questioned whether I was migrating the data accurately. Part of my discomfort may have stemmed from the constraints of the small Postgres interface within Udacity, which hindered my ability to easily visualize a substantial amount of data on the screen. Nonetheless, I am confident that this discomfort will quickly fade as I gain more SQL experience.
